10. Einstellungen


10.1 Pluginmanager
Menü -> Settings -> Settings -> Pluginmanager
Toolbar -> Settings -> Pluginmanager

Settings-Pluginmanager
Available Plugins: Hier werden alle Plugins mit der jeweiligen Version aufgelistet, die zur Zeit installiert sind (Hier der Block-Generator). Rechts davon wird eine kurze Beschreibung des markierten Plugins angezeigt. Geladene Plugins haben einen grünen Hacken  , vor nicht geladene befindet sich ein rotes Stopp .
Load: Hiermit wird das unter Available Plugins selektierte Plugin geladen. Das bedeutet, das die Funktionalität des Plugins im Editor oder von anderen Plugins verwendet werden kann. Diese sind im allgemeinen über einen Button oder über das Menü des Anwendungsfensters erreichbar.
Install: Hier öffnet sich ein Dateiauswahldialog, in dem man das zu installierende Plugin auswählen kann. Um es dann zu benutzen muss es noch mit Load geladen werden. Siehe dazu auch: 'alternative Installation'

Uninstall:

 Deinstalliert das markierte Plugin. Die Datei wird nicht von der Festplatte gelöscht. Um auf die Funktionalität des Plugins erneut zuzugreifen zu können, kann es über Install erneut installiert werden.
Unload: Unload deaktiviert das selektierte Plugin, das heißt, es kann nicht mehr verwendet werden. Es wird aber nicht deinstalliert! Durch ein Load kann es wieder geladen werden.
Autosearch for Plugins: Falls diese Option gesetzt ist, wird bei jedem Start der Ordner Path nach gültigen Plugins durchsucht und diese installiert.
Path: Siehe Autosearch for Plugins
Load plugins automatically: Diese Option zeigt an, ob die Plugins, die per Autosearch in Path gefunden werden, nicht nur installiert, sondern auch geladen werden sollen.

Alternative Installation:
Ein Plugin kann auch installiert werden, wenn die entsprechende Datei (.dll oder .so) in das Verzeichnis, das mit Path angegeben ist, kopiert wird (Standartmäßig: /Plugins). Das Plugin wird dann beim nächsten Programmstart entsprechend den Optionen Autosearch for Plugins bzw. Load plugins automatically installiert bzw. geladen.

Achtung:
Falls der Pfad zum Plugin oder der Dateiname nicht-ASCII-Zeichen enthält, kann das Pluginsystem unvorhergesehen reagieren.

10.2 Look & Feel


Look & Feel Settings

Hier können sie den Stil für Blacksun festlegen. Normalerweise wählt Blacksun einen Stil aus der zu ihrer Arbeitsumgebung passt. Sollte dies nicht der Fall sein oder sollte er ihnen nicht gefallen können sie hier einen anderen festlegen. Falls sie einen zusätzlichen Qt4 Stil installiert haben können sie diesen hier auswählen.

10.2 Look & Feel


Mouse

Hier können sie die Einstellungen für die Maus jeweils für die 3D und 2D-Ansicht einstellen. Die einträge in en Comboboxen haben folgende Bedeutung:

Selected Action: Es wird die unter Actionmode ausgewählte Aktion durchgeführt.
Move: Die Kamera wird verschoben.
Add to Selection: Falls unter Actionmouse Select ausgewählt wurde werden neu markierte Objekte zur aktullen Selektion hinzugefügt.

Remove from Selection:

 Falls unter Actionmouse Select ausgewählt wurde werden neu markierte Objekte von der aktuellen Selektion entfernt.
Rotate: Die Kamera wird gedreht.
Nothing: Es passiert nichts.



10.3 Renderer
Menü -> Settings -> Settings -> Renderer
Toolbar -> Settings

10.3.1 Optionen
View
Grid size (partitions per decade): Die Anzahl der Unterteilungen des Gitters je Dekade. Um eine Unterteilung in 0.2er Schritten zu Erhalten, wählt man beispielsweise 5.
Line width: Die Liniendicke für das Gitter, Wireframe-Modus, ...
Point size: Die Punktdicke für Wireframe-, Point-Modus, ...
Normal scaling: Der Skalierungsfaktor für die Darstellung der Normalen. In der Regel ist dieser Faktor gleich der Länge der gezeichneten Linie, da die Normalenvektoren normalisiert sind (bzw. sein sollten)
Selection box scaling: Der Skalierungsfaktor für die Selektionsbox, die nach dem Selektieren von Szenendaten dargestellt wird
Wireframe overlay scaling: Der Skalierungsfaktor für den Wireframe-Overlay-Modus. Er gibt an, inwieweit der Wireframe-Anteil skaliert werden soll. Sollten die Linien nicht gut erkennbar sein, sollte man den Faktor minimal erhöhen.
Mousewhel zoomspeed: Die Geschwindigkeit des Zooms, der mit dem Mausrad verbunden ist
Field of view (in degrees): Das Sichtfeld der perspektivischen (3D-)Kamera in Grad
Near clipping plane: Die Near Clipping Plane. Alle Vertices, die von der Kamera einen kleineren Abstand als dieser Wert haben, werden nicht dargestellt.
Far clipping plane: Die Far Clipping Plane. Alle Vertices, die von der Kamera einen größeren Abstand als dieser Wert haben, werden nicht dargestellt.
Culling (Frustum): Aktiviert/Deaktivert das Frustum-Culling
Line smoothing: Aktiviert/Deaktivert das Glätten der Linien
Point smoothing: Aktiviert/Deaktivert das Glätten der Punkte
Polygon smoothing: Aktiviert/Deaktivert das Glätten der Polygone (auf einigen Grafikkartenführt dies aber zu Darstellungsfehlern!!)




10.3.2 Farbschema


User Interface Colors
Line color: Farbe der Linien im Wireframe-Modus für unselektierte Faces
Selected line color: Farbe der Linien im Wireframe-Modus für selektierte Faces
Point color (Vertex, ...): Farbe der Punkte im Wireframe- und Punkt-Modus für unselektierte Faces
Selected point color (Vertex, ...): Farbe der Punkte im Wireframe- und Punkt-Modus für selektierte Faces
Freezed object's color: Farbe der Linien und Punkte im Wireframe- und Punk-Modus, falls das Face gesperrt ist
Normal color: Farbe der grafisch darzustellenden Normalenvektoren
Selection box color: Farbe der Linien der Selektionsbox
Selection rubber band color: Farbe des Selektionsrechtecks bei der Selektierung per Maus
Overpainting color: Farbe der Szenenübermalenden Objekte (Schrift, ...)
Orthogonal viewport color: Hintergrundfarbe für die orthogonale (2D-)Ansicht
Perspective viewport color: Hintergrundfarbe für die perspektivische (3D-)Ansicht
Major gridlines color: Farbe für die Major-Gitterlinien
Minor gridlines color: Farbe für die Minor-Gitterlinien
Load Color Scheme: Lädt alle Farben aus einem Datei (das Farbschema)
Save Color Scheme: Speichert alle Farben in eine Datei (Das Farbschema)
Reset default colors: Setzt alle Farben zurück zu ihrem Standardwerten


10.4 Logfenster
Log-Fenster des Editors
Während der Ausführung des Programms werden alle wichtigen Vorgänge in der Log-Datei gespeichert. Hierbei wird gesichert um welche Art von Log-Eintrag es sich handelt und dazu eine kurze Nachricht.

Die Log-Einträge kann man sich im Editor selbst,
DEBUG -> Log Window
oder in html Form in einem Browser
Installationsverzeichnis -> Logfile.html
anzeigen lassen.
Log-Datei in html
Es gibt 4 verschiedene Arten von Log-Einträgen, die zusätzlich durch verschiedene Farben unterschieden werden:

ERROR Ein Fehler ist während der Aktion aufgetreten Rot
OK Die Aktion ist ordnungsgemäß abgelaufen Grün
WARNING Die Aktion wurde mit Abweichungen ausgeführt Orange
OTHER sonstige Informationen/Hinweise Schwarz
